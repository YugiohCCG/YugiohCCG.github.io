# Extensive YGO Omega Lua Audit - 2026-07-17

## Outcome

The complete custom corpus of 481 standalone Lua scripts was mechanically tested against the official YGO Omega Lua API and helper corpus in `tmp/omega_scripts`. The latest 27 generated scripts were also reviewed individually against `src/data/cards.json`, the Omega ID mapping, public and installed DB rows/messages, and the closest official effect implementations.

- 23 of the latest 27 scripts required and received fixes.
- 3 of the latest 27 passed without a Lua logic change.
- 1 card, `Devotee of Fire`, still needs a manual text/ruling decision.
- 21 older scripts received additional corpus-level compatibility or structural corrections. One of those changes was whitespace-only; the other 20 addressed real engine/API or registration risks.
- Public, installed Omega, ZIP, and `dist` mirrors were rebuilt and verified after the final fixes.

This was a static, registration-smoke, DB, and artifact-parity audit. It does not replace live-duel confirmation of timing prompts, player choices, or copied-effect behavior.

## Additional Corpus-Level Fixes

The all-script API/structure sweep found issues outside the newest batch:

- `c259688514.lua`, `c259724129.lua`, and `c259937399.lua` had nested duplicate `initial_effect` definitions. In Omega, the outer function returned before registering the later effects. Each now has exactly one initializer.
- `c259937399.lua` used an unsupported generic `EFFECT_EXTRA_MATERIAL` pattern. It now follows Omega's official hand-based Extra Link Material registration pattern.
- `c250339529.lua` and `c259920959.lua` called nonexistent `aux.AddFusionProcMixN`; both now use the official repeated-material Fusion helper.
- `c210711859.lua` requested unsupported chain-info fields and called `GetHandler()` on a numeric reason value; it now uses the official triggering effect/player fields and effect handler.
- `c226827770.lua` used invalid `AnnounceCard` and operation-info arguments; both now match Omega's official announcement form.
- `c255686110.lua` and `c256469525.lua` used nonexistent `RESETS_STANDARD_DISABLE`.
